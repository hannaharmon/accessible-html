Why Accessibility?

There are different users of different abilities and backgrounds. To make webpages accessible for all kinds of users is essential.

How Accessibility?

A good rule of thumb for accessible web pages is to follow POUR (Perceivable, Operable, Understandable, and Robust):

  • Perceivable: Information and user interface must be presentable in ways users can perceive (e.g., text alternatives for images, captions for audio).
  • Operable: User interface components and navigation must be operable (e.g., keyboard accessibility, sufficient time for interactions).
  • Understandable: Information and the operation of the user interface must be understandable (e.g., clear language, predictable navigation, error prevention).
  • Robust: Content must be robust enough to be interpreted reliably by a wide variety of user agents, including assistive technologies (e.g., HTML standards, browser compatibility)
